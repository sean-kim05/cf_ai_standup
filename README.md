# cf_ai_standup

An AI-powered daily standup bot built on Cloudflare. Type what you worked on in plain English and get a structured standup (Yesterday / Today / Blockers) generated by Llama 3.3.

**Live Demo:** https://briefly.skim8705.workers.dev

---

## Features

- **AI standup formatting** ‚Äî paste messy notes, get a clean Yesterday / Today / Blockers standup
- **Weekly summary** ‚Äî one-click AI summary of your last 7 standups
- **Persistent history** ‚Äî standup history stored per browser via Durable Objects
- **Copy to clipboard** ‚Äî copy your formatted standup instantly
- **Mobile responsive** ‚Äî works on phone or desktop

---

## Tech Stack

| Component | Technology |
|---|---|
| LLM | Llama 3.3 via Cloudflare Workers AI |
| Workflow / API | Cloudflare Worker |
| User Input / UI | Chat-style UI hosted on Cloudflare Pages |
| Memory / State | Cloudflare Durable Objects (per-user standup history) |

### Cloudflare AI Components
1. **LLM** ‚Äî `@cf/meta/llama-3.3-70b-instruct-fp8-fast` via Workers AI for standup formatting and weekly summaries
2. **Workflow** ‚Äî Cloudflare Worker handles all API routing (`POST /api/standup`, `GET /api/history/:userId`, `POST /api/weekly/:userId`)
3. **User input** ‚Äî Single-page HTML/CSS/JS UI deployed on Cloudflare Pages; userId stored in localStorage for session persistence
4. **Memory/state** ‚Äî Durable Object (`StandupMemory`) stores up to 100 standup entries per user, keyed by userId

## Local Development

### Prerequisites
- Node.js 18+
- [Wrangler CLI](https://developers.cloudflare.com/workers/wrangler/install-and-update/) (`npm install -g wrangler`)
- Cloudflare account (free)

### Setup

```bash
git clone https://github.com/sean-kim05/cf_ai_standup.git
cd cf_ai_standup
npm install
```

### Run the Worker locally

```bash
npm run dev
# Worker runs on http://localhost:8787
```

### Open the frontend

Open `public/index.html` directly in your browser. It points to `http://localhost:8787` by default.

> Note: Workers AI does not run locally ‚Äî `wrangler dev` proxies AI calls to Cloudflare's remote infrastructure.

---

## Deployment

### Deploy the Worker

```bash
wrangler login    # authenticate with Cloudflare
npm run deploy    # deploys worker + Durable Object
```

### Deploy the Frontend to Cloudflare Pages

1. Update `API_URL` in `public/index.html` to your deployed worker URL
2. Push to GitHub
3. In the Cloudflare dashboard ‚Üí Pages ‚Üí Connect to Git ‚Üí select this repo
4. Set build output directory to `public`

## API Reference

| Endpoint | Method | Body | Description |
|---|---|---|---|
| `/api/standup` | POST | `{ userId, message }` | Format message into standup, save to history |
| `/api/history/:userId` | GET | ‚Äî | Return last 7 standups |
| `/api/weekly/:userId` | POST | ‚Äî | Generate weekly summary from last 7 standups |
